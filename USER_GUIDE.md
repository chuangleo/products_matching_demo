# 電商商品比對系統使用手冊

## 摘要

本系統為一基於人工智慧技術的跨電商平台商品比對工具，旨在自動化辨識 MOMO 購物網與 PChome 購物平台上的相同商品，並提供價格比較功能。系統整合網頁爬蟲技術、深度學習語意模型（E5-Large Multilingual）及 Google Gemini 大型語言模型，實現高準確度的商品配對與智慧驗證機制。

## 1. 系統概述

### 1.1 研究背景與動機

隨著電子商務市場的蓬勃發展，消費者面臨多個購物平台的商品價格比較需求。然而，由於各平台商品資訊格式不一、命名規則差異，人工比對耗時且容易出錯。本系統提供自動化解決方案，協助使用者快速找出不同平台上的相同商品並進行價格比較。

### 1.2 系統目標

1. **自動化商品資料擷取**：透過網頁爬蟲技術自動蒐集目標電商平台的商品資訊
2. **智慧商品配對**：運用深度學習模型進行語意相似度分析，識別相同商品
3. **AI 輔助驗證**：整合大型語言模型進行二次驗證，提升配對準確度
4. **視覺化介面**：提供直觀的使用者介面，降低技術門檻

### 1.3 核心功能

- **商品資料擷取**：支援 MOMO 購物網與 PChome 購物平台的商品資訊爬取
- **語意相似度計算**：採用多語言 E5-Large 模型進行商品標題的語意編碼與相似度運算
- **智慧配對驗證**：運用 Gemini AI 模型分析商品規格、品牌、型號等特徵，進行精準配對
- **價格比較展示**：以視覺化方式呈現配對結果與價格資訊

## 2. 系統架構與技術規格

### 2.1 系統架構

本系統採用三層式架構設計：

1. **資料擷取層（Data Acquisition Layer）**
   - 使用 Selenium WebDriver 進行動態網頁內容擷取
   - 支援分頁瀏覽與商品資訊批次處理
   - 實作反爬蟲機制規避與錯誤處理機制

2. **資料處理與分析層（Processing & Analysis Layer）**
   - 採用 E5-Large Multilingual 預訓練模型進行文本編碼
   - 實作餘弦相似度（Cosine Similarity）計算演算法
   - 整合 Google Gemini API 進行智慧驗證

3. **呈現層（Presentation Layer）**
   - 基於 Streamlit 框架開發互動式網頁介面
   - 提供即時進度回饋與結果視覺化
   - 支援響應式設計與多裝置存取

- **程式語言**：Python 3.10 或以上版本
- **套件管理**：uv (Astral)
- **網頁框架**：Streamlit 1.x
- **作業系統**：Windows / Linux / macOS

#### 2.2.2 核心套件

#### 4.2.4 自動配對流程

系統採用二階段配對機制：

**第一階段：語意相似度篩選**

1. **文本預處理**
   - MOMO 商品標題加上前綴：`"query: "`
   - PChome 商品標題加上前綴：`"passage: "`

2. **向量編碼**
   - 使用 E5-Large Multilingual 模型
   - 將商品標題轉換為 1024 維向量

3. **相似度計算**
   - 採用餘弦相似度（Cosine Similarity）
   - 公式：$\text{similarity} = \frac{\mathbf{A} \cdot \mathbf{B}}{||\mathbf{A}|| \times ||\mathbf{B}||}$
   - 範圍：0.0 ~ 1.0

4. **候選商品篩選**
   - 門檻值：0.739465（系統預設）
   - 篩選相似度 ≥ 門檻值的 PChome 商品
   - 依相似度由高至低排序

**第二階段：AI 智慧驗證**

對第一階段篩選出的候選商品，逐一進行 AI 驗證：

1. **輸入資料**
   - MOMO 商品標題
   - PChome 商品標題
   - 第一階段相似度分數

2. **Gemini AI 分析**
   - 品牌與型號比對
   - 規格差異檢測
   - 顏色變體識別
   - 組合包與單品區分
   - 原廠與副廠判別

3. **輸出結果**
   - `is_match`：配對結果（true/false）
   - `confidence`：信心水準（high/medium/low）
   - `reasoning`：判斷理由（繁體中文）

### 5.2 語意相似度計算

#### 5.2.1 E5 模型架構

採用 **intfloat/multilingual-e5-large** 預訓練模型：

**模型規格**：
- 基礎架構：Transformer Encoder
- 參數量：560M
- 最大序列長度：512 tokens
- 輸出維度：1024

**訓練方式**：
- 對比學習（Contrastive Learning）
- 多語言語料庫（100+ 語言）
- 指令調校（Instruction Tuning）

#### 5.2.2 文本編碼流程

```python
# MOMO 商品編碼
momo_text = "query: " + product_title
momo_embedding = model.encode(momo_text)

# PChome 商品編碼
pchome_text = "passage: " + product_title
pchome_embedding = model.encode(pchome_text)
```

**前綴設計理由**：
- `query:`：表示查詢端（MOMO 商品）
- `passage:`：表示被查詢端（PChome 商品）
- 符合 E5 模型的訓練範式，提升檢索效能

#### 5.2.3 相似度計算公式

採用餘弦相似度（Cosine Similarity）：

$$
\text{similarity}(A, B) = \frac{\sum_{i=1}^{n} A_i \times B_i}{\sqrt{\sum_{i=1}^{n} A_i^2} \times \sqrt{\sum_{i=1}^{n} B_i^2}}
$$

其中：
- $A$：MOMO 商品向量
- $B$：PChome 商品向量
- $n$：向量維度（1024）

**相似度解釋**：
- 1.0：完全相同
- 0.8-1.0：高度相似
- 0.6-0.8：中度相似
- 0.0-0.6：低度相似

### 5.3 AI 驗證機制

#### 5.3.1 Gemini API 整合

**模型版本**：Gemini Pro

**輸入格式**：
```json
{
  "momo_title": "商品標題",
  "pchome_title": "商品標題",
  "similarity_score": 0.8523
}
```

**輸出格式**：
```json
{
  "is_match": true,
  "confidence": "high",
  "reasoning": "品牌與型號相同，僅顏色不同"
}
```

#### 5.3.2 配對規則

系統實作以下商品配對邏輯：

**必要條件**（全部符合視為相同商品）：
1. 品牌名稱一致
2. 產品型號一致
3. 主要規格一致（如容量、尺寸）

**排除條件**（任一符合視為不同商品）：
1. 組合包 vs 單品
2. 原廠 vs 副廠/相容品
3. 限量版 vs 一般版
4. 福利品 vs 全新品

**容許差異**：
- 顏色變體
- 語言差異（中文 vs 英文品牌名）
- 標點符號差異

#### 5.3.3 信心水準評估

| 信心水準 | 判斷標準 |
|---------|---------|
| High | 品牌、型號、規格完全一致 |
| Medium | 品牌、型號一致，規格部分相同 |
| Low | 僅品牌一致或資訊不足 |

### 5.4 結果視覺化

#### 5.4.1 商品卡片設計

**配對成功卡片（綠色）**：
- 邊框顏色：`#48bb78`（綠色）
- 背景顏色：`#f0fff4`（淺綠）
- 狀態標示：✅ 配對成功

**配對失敗卡片（紅色）**：
- 邊框顏色：`#f56565`（紅色）
- 背景顏色：`#fff5f5`（淺紅）
- 狀態標示：❌ 未配對

#### 5.4.2 資訊呈現元素

每張商品卡片包含：
1. 平台標籤（PChome）
2. 商品圖片（120x120px）
3. 商品標題
4. 價格資訊（NT$）
5. 相似度分數（四位小數）
6. AI 判斷理由（繁體中文）
7. 商品連結按鈕

## 6. 常見問題與解決方案

### 6.1 系統安裝與設定

#### Q1：Python 版本相容性問題

**問題描述**：系統要求 Python 3.10+，但已安裝較舊版本

**解決方案**：
1. 前往 Python 官網下載最新版本
2. 解除安裝舊版本（Windows：控制台 > 程式集）
3. 安裝新版本並勾選「Add Python to PATH」
4. 重新開啟終端機驗證版本

#### Q2：uv 套件管理工具無法安裝

**問題描述**：執行安裝指令時出現權限錯誤

**解決方案**：
- Windows：以系統管理員身分執行 PowerShell
- Linux/macOS：使用 `sudo` 權限或檢查 shell 設定檔

#### Q3：Gemini API 金鑰設定錯誤

**錯誤訊息**：`請設定 Gemini API Key 才能使用 AI 驗證功能`

**解決方案**：
1. 檢查 `.env` 檔案是否存在於專案根目錄
2. 確認檔案內容格式：`GEMINI_API_KEY=your_key`
3. 確認 API 金鑰有效性（至 Google AI Studio 確認）
4. 重新啟動系統使環境變數生效

### 6.2 商品搜尋與擷取

#### Q4：爬蟲執行失敗或逾時

**可能原因**：
- 網路連線不穩定
- 購物網站回應緩慢
- Chrome 瀏覽器驅動版本不符

**解決方案**：
1. 檢查網路連線狀態
2. 重新執行搜尋
3. 降低搜尋數量（如改為 30 件）
4. 更新 Chrome 瀏覽器至最新版本

#### Q5：搜尋結果商品數量少於設定值

**問題說明**：設定搜尋 50 件，實際只擷取 30 件

**原因分析**：
- 購物網站該關鍵字的商品總數不足
- 系統自動去除重複商品（相同 SKU）
- 部分商品資訊不完整被系統過濾

**處理建議**：
此為正常現象，不影響系統功能。系統會保留所有有效商品資料。

#### Q6：商品圖片無法顯示

**問題類型**：
1. 圖片載入失敗（顯示「無法載入圖片」）
2. 圖片顯示為官方標籤而非商品圖

**解決方案**：
- 類型 1：網路問題或圖片連結失效，不影響配對功能
- 類型 2：系統已實作過濾機制，建議重新執行爬蟲更新資料

### 6.3 商品配對

#### Q7：高相似度商品未通過 AI 驗證

**情況範例**：相似度 0.85，但 AI 判斷為不同商品

**原因說明**：
- 相似度僅反映標題文字的語意相近程度
- AI 驗證會進一步檢查品牌、型號、規格等細節
- 可能存在「組合包 vs 單品」等實質差異

**系統設計邏輯**：
二階段驗證機制確保配對準確性，降低誤配率。

#### Q8：相同商品被判定為不同商品

**可能原因**：
1. 標題描述差異過大
2. 一方為特殊版本（如限量版、福利品）
3. 規格資訊不完整導致 AI 無法判斷

**建議做法**：
1. 檢視 AI 判斷理由
2. 點擊商品連結人工確認
3. 若確認為誤判，可記錄為系統改進參考

#### Q9：配對時間過長

**時間標準**：
- 第一階段（語意計算）：5-10 秒
- 第二階段（AI 驗證）：2-3 秒/商品

**優化建議**：
1. 確保網路連線品質
2. 若候選商品過多（>20 件），考慮提高相似度門檻
3. 關閉其他網路密集型應用程式

### 6.4 系統運作

#### Q10：系統啟動後瀏覽器未自動開啟

**解決方案**：
1. 手動開啟瀏覽器
2. 輸入網址：`http://localhost:8501`
3. 檢查終端機是否顯示正確的 URL

#### Q11：系統顯示「找不到模型路徑」

**錯誤訊息**：`找不到模型路徑：models/...`

**排查步驟**：
1. 確認 `models/` 目錄是否存在
2. 檢查目錄內是否包含完整模型檔案
3. 驗證檔案權限（可讀取）
4. 確認檔案路徑設定正確

**緊急處理**：
聯絡系統管理員重新部署模型檔案。

#### Q12：CSV 檔案為空或損毀

**問題表現**：
- 系統顯示「目前系統中還沒有任何商品資料」
- 或選擇類別後無商品可選

**解決方案**：
1. 執行商品搜尋功能重新建立資料
2. 檢查 CSV 檔案是否存在且可讀取
3. 必要時刪除損毀的 CSV 檔案，系統將自動重建

### 6.5 效能調校

#### Q13：如何提升系統效能？

**建議措施**：
1. **硬體層面**
   - 使用 SSD 固態硬碟
   - 配置 16GB 以上記憶體
   - 確保穩定的網路頻寬

2. **軟體層面**
   - 關閉不必要的背景程式
   - 定期清理 CSV 資料（移除過時商品）
   - 限制搜尋數量（建議 50 件）

3. **參數調整**
   - 提高相似度門檻（減少 AI 驗證次數）
   - 限制候選商品數量上限（如 100 件）

### 6.6 資料管理

#### Q14：如何清除舊資料？

**操作步驟**：
1. 關閉系統
2. 刪除 `momo.csv` 和 `pchome.csv`
3. 重新啟動系統
4. 執行商品搜尋重建資料庫

**注意事項**：
建議定期備份 CSV 檔案，避免資料遺失。

#### Q15：能否匯出配對結果？

**現況說明**：
系統目前不支援配對結果匯出功能。

**替代方案**：
1. 使用瀏覽器內建的網頁擷取功能
2. 手動記錄配對商品資訊
3. 針對重要商品建立書籤

## 7. 結論

本系統整合先進的網頁爬蟲技術、深度學習語意模型及大型語言模型，提供自動化的跨平台商品配對解決方案。透過二階段驗證機制，系統在保持高召回率的同時，確保配對精確度。使用者介面設計直觀，降低技術門檻，使一般消費者亦能輕鬆使用。

### 7.1 系統優勢

1. **自動化流程**：從資料擷取到配對驗證全程自動化
2. **高準確度**：結合語意模型與 AI 驗證的二階段機制
3. **易用性**：圖形化介面，無需程式設計背景
4. **擴充性**：模組化設計，易於擴充至其他電商平台

### 7.2 應用價值

- **學術研究**：提供電商商品配對的實作範例
- **消費者**：協助價格比較，提升購物決策品質
- **電商業者**：競品分析與價格監控工具

4. 開發價格變動通知機制
